# MyTodos团队待办App应用 - 提示词
## 需求提示词
### 0100
需要写一个需求规格说明书，包括功能需求、非功能需求、界面需求、数据库需求等。
一句话需求：
一个面向 2-5 人小团队的移动端待办事项协作应用。使用 Rust + Tauri + VUE3 开发，通过 Gitee 代码片段 API 存储数据，无需自建后端数据存储。

### 0200

首页界面
应该是选择登录用户，输入6位口令，点击登录按钮。
右上角有一个管理员入口，输入6位口令，点击后进入管理员界面。
管理员界面
1、有添加用户、删除用户、修改用户权限等功能。
父母或学生用户界面：
1、可以查看所有待办事项。
2、可以添加、删除、修改待办事项。
