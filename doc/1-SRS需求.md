# MyTodos 团队待办 App — 需求规格说明书（SRS）

## 1. 引言

### 1.1 文档目的

本文档用于描述"MyTodos 团队待办 App"的需求规格，包括功能需求、非功能需求、界面需求、数据库/数据需求与外部接口需求，为产品设计、开发实现、验收测试提供依据。

### 1.2 产品一句话需求

一个面向 2-5 人小团队的移动端待办事项协作应用。使用 Rust + Tauri + Vue 3 开发，通过 Gitee 代码片段（Gist）API 存储数据，无需自建后端数据存储。

### 1.3 范围与边界

- 目标用户：2-5 人的小团队，如家庭场景，包含三种角色：管理员、家长、学生。
- 目标端：移动端优先（Android/iOS），同时可编译桌面端（Windows/macOS/Linux）方便调试，共用一套代码。
- 数据存储：远端使用 Gitee 代码片段 API，需联网使用，实时读写 Gitee 数据。
- 不在范围内：
  - 自建服务器、传统后端数据库、IM 即时通讯。
  - 复杂的组织/权限体系（例如多部门、多项目层级、细粒度 ACL）。
  - 企业级审计、SLA、可用性承诺。
  - 本地数据缓存、离线队列、离线操作。
  - 评论功能、消息通知/提醒功能。

### 1.4 术语与缩略语

- **Workspace（工作区）**：一个团队空间，包含一组成员及其共享的待办数据集合。
- **Task（任务）**：待办事项。
- **Gist**：Gitee 代码片段（本文统一称"代码片段"），用于承载 JSON 数据文件。
- **PAT**：Personal Access Token（个人访问令牌），用于调用 Gitee API。
- **角色**：
  - **管理员**：负责工作区创建与配置、成员管理、标签管理。
  - **家长**：负责任务的创建、编辑、删除。
  - **学生**：负责查看任务、处理任务（标记进行中/完成）。

### 1.5 参考与约束

- 技术栈：Rust + Tauri 2 + Vue 3。
- 外部依赖：Gitee API（v5），使用时必须保持网络连接。

## 2. 总体描述

### 2.1 产品视角

MyTodos 是一个无需自建后端的协作待办应用。所有团队数据以结构化 JSON 文件形式存储在 Gitee 代码片段中；客户端通过 PAT 直接调用 Gitee API 完成数据读写，为纯在线应用，不可脱机使用。

### 2.2 用户角色与特征

**管理员**

- 首次进入时创建或配置工作区。
- 管理工作区基础设置（名称、成员等）。
- 管理标签（创建、编辑、删除标签）。
- **不可**创建、编辑、删除、完成任务。

**家长**

- 创建、编辑、删除任务。
- 为任务指定优先级、截止日期、指派人、标签。
- 查看所有任务列表，支持筛选与搜索。
- **不可**将任务标记为进行中或完成。

**学生**

- 查看分配给自己的任务（待办/进行中）。
- 将任务标记为**进行中**，记录开始时间。
- 将任务标记为**已完成**，记录完成时间。
- **不可**创建、编辑、删除任务。

### 2.3 运行环境

- 移动端：Android / iOS（Tauri Mobile）。
- PC 端：Windows / macOS / Linux。
- 设备能力：
  - 必须：网络访问（HTTPS）。
  - 可选：本地通知（到期提醒，后期版本考虑）。

### 2.4 设计与实现约束

- 不引入自建后端：所有协作数据只能通过 Gitee API 存取。
- 纯在线模式：所有读写操作直接请求 Gitee API，不设本地缓存层。
- 多端并发写入：需要处理冲突（乐观并发控制，基于 revision 检测冲突）。

### 2.5 关键假设

- 整个 APP 内置 Gitee 帐户，在编译时通过 `.env` 读取 Gitee 帐号信息。
- 团队规模 2-5 人，任务量中小规模（建议单工作区活跃任务不超过 200 条）。
- 所有设备使用同一内置 Gitee 帐号访问代码片段，权限控制由应用层角色机制实现。

## 3. 功能需求

### 3.1 账户与工作区

**FR-001 Gitee 凭证配置**

- 应用内置 Gitee PAT，在编译时通过 `.env` 文件注入。
- PAT 随每次 API 请求携带，用于认证。
- PAT 由管理员在构建/部署前配置，终端用户不可见、不可修改。

**FR-002 首次进入与角色选择**

- 用户首次打开应用时：
  - 选择角色（管理员 / 家长 / 学生）。
  - 输入 6 位数字口令（由管理员预先分发），验证通过后进入对应界面。
- 角色和口令保存在本地系统安全存储中，后续启动自动读取，无需重复选择。
- 各角色进入后的目标页面：
  - **管理员** → UI-002 工作区配置页。
  - **家长** → 选择工作区 → UI-101 任务列表页。
  - **学生** → 选择工作区 → UI-101 任务列表页。

**FR-003 创建工作区**

- 管理员可创建工作区，填写：
  - 工作区名称（必填），例如：一中实验初中、软件谷小学等。
  - 工作区描述（可选）。
- 创建工作区时，应用应创建或绑定一个 Gitee 代码片段作为远端存储，并生成工作区标识：
  - workspaceId（UUID）
  - gistId（代码片段 ID）
  - workspaceKey（可选：本地加密口令，用于对远端 JSON 加密）

**FR-004 加入/切换工作区**

- 成员在启动时需要选择工作区。
- 用户可在多个工作区之间切换。

**FR-005 成员管理**

- 管理员可维护成员列表（2-5 人）：
  - 添加成员：记录成员显示名、角色（管理员 / 家长 / 学生）。
  - 编辑成员信息。
  - 移除成员。

**FR-006 标签管理**

- 管理员可在工作区设置中管理标签：
  - 创建标签：标签名称（必填，1-20 字）、颜色标识（可选）。
  - 编辑标签：修改名称或颜色。
  - 删除标签：删除后所有任务中该标签的关联同步移除。
- 家长在创建/编辑任务时可选择已存在的标签，**不可**创建或管理标签。
- 标签数量上限：20 个/工作区。

### 3.2 任务管理

> 注：以下操作均在网络连接正常的前提下进行，所有变更实时同步至 Gitee 代码片段。

**FR-101 创建任务**
> 权限：仅家长角色。

- 创建任务字段至少包含：
  - 标题（必填，1-80 字）
  - 详情描述（可选，支持换行，1-256 字）
  - 状态（必填，默认"待办"）：待办 / 进行中 / 已完成
  - 截止日期时间（必填）
  - 优先级（必填，默认"中"）：低 / 中 / 高
  - 指派人（必填，只支持 1 个成员）
  - 标签（可选，从工作区已有标签中选择，可多选）

**FR-102 编辑任务**
> 权限：仅家长角色。

- 家长可编辑任务的所有字段（标题、描述、状态、截止日期、优先级、指派人、标签）。
- 编辑后更新 updatedAt/updatedBy 字段。

**FR-103 处理任务（进行中/完成/恢复）**
> 权限：家长和学生角色。管理员不可操作。

- 学生可将分配给自己的任务标记为**进行中**：
  - 状态变为"进行中"。
  - 记录任务开始时间（startedAt）。
- 学生可将分配给自己的任务标记为**已完成**：
  - 状态变为"已完成"。
  - 记录任务完成时间（completedAt/completedBy）。
- 家长或学生可将进行中/已完成的任务恢复到**待办**状态：
  - 清除 startedAt、completedAt、completedBy 字段。

**FR-104 删除任务**
> 权限：仅家长角色。

- 家长可删除任务；删除为逻辑删除，标记为删除状态（deletedAt/deletedBy）。
- 已删除的任务不在列表中展示，但数据保留在远端。
- 管理员和学生不可删除任务。

**FR-105 任务列表与筛选**

家长视图：
- 显示所有未删除的任务。
- 筛选条件：
  - 按状态筛选：待办 / 进行中 / 已完成
  - 按指派人筛选
  - 按截止日期筛选：今天 / 本周 / 逾期 / 无截止
  - 按标签筛选（多选）
- 默认排序：逾期优先 → 截止时间升序 → 创建时间降序。

学生视图：
- 默认只显示分配给自己的、状态为"待办"或"进行中"的任务。
- 右上角过滤切换：
  - **进行中**：显示待办 + 进行中的任务。
  - **完成**：只显示已完成的任务。
- 默认排序：逾期优先 → 截止时间升序 → 创建时间降序。

**FR-106 任务搜索**

- 提供关键词搜索功能（所有角色可用）：
  - 搜索范围：任务标题 + 详情描述。
  - 搜索结果受当前角色的视图权限约束：
    - 家长：搜索所有未删除任务。
    - 学生：仅搜索分配给自己且状态为待办/进行中的任务。
  - 搜索结果保持当前筛选条件与排序规则。
  - 无结果时显示友好的空状态提示。

## 4. 外部接口需求

### 4.1 与 Gitee API 的交互

- 认证方式：PAT（access_token）随请求携带。
- 主要能力：
  - 创建代码片段（用于初始化工作区）
  - 获取代码片段内容（拉取远端数据）
  - 更新代码片段内容（推送变更）
- 失败处理：
  - 401/403：提示凭证无效或权限不足，引导重新配置 PAT。
  - 404：代码片段不存在，引导重新加入或迁移。
  - 429/5xx：指数退避重试，重试失败后提示用户稍后再试。

### 4.2 与系统能力的交互

- 本地通知：用于截止提醒（可选，后期版本考虑）。
- 系统安全存储：用于保存 PAT、workspaceKey、用户角色和口令等敏感信息。
- 文件系统：用于导入/导出备份文件。

## 5. 界面需求（UI/UX）

### 5.1 全局原则

- 单手可操作：核心操作（新增、完成、切换状态）在 2 次点击内完成。
- 信息层级清晰：列表聚焦任务标题与关键状态（截止/指派/优先级）。
- 网络状态明确：网络请求状态（加载中/成功/失败）有可见提示。

### 5.2 页面清单与要点

**UI-001 首次进入/引导页**

- 首次打开应用时显示角色选择界面。
- 用户选择角色（管理员 / 家长 / 学生）后输入 6 位数字口令。
- 验证通过后保存角色和口令到本地安全存储。
- 后续启动直接从本地读取，跳过引导页。
- 各角色进入后的目标页面：
  - **管理员** → UI-002 工作区配置页。
  - **家长** → 选择工作区 → UI-101 任务列表页（可发布与管理事项）。
  - **学生** → 选择工作区 → UI-101 任务列表页（只可查看与处理事项）。

**UI-002 工作区列表/配置页**

- 管理员视图：
  - 展示已创建的工作区列表。
  - 支持"创建工作区"、"编辑工作区"、"管理成员"、"管理标签"入口。
- 家长/学生视图：
  - 展示已加入的工作区列表。
  - 支持"加入工作区（输入 gistId）"。

**UI-101 任务列表页（主页面）**

- 顶部：工作区名称 + 网络状态指示。
- 搜索栏：关键词搜索入口（所有角色可见）。
- 中部：任务列表（按筛选/排序展示）。
- 底部/悬浮按钮：新增任务（仅家长角色可见）。
- 列表项展示：
  - 标题、截止时间、指派人名称、优先级标识、标签色块。
  - 复选框或滑动操作：完成任务、进行中标记（仅家长/学生可用）。
- 筛选栏：
  - 家长：状态、指派人、截止日期、标签筛选。
  - 学生：进行中/完成切换。

**UI-102 任务详情页**

- 展示完整字段：标题、描述、状态、截止时间、优先级、指派人、标签。
- 家长角色：支持编辑入口与所有字段修改。
- 学生角色：支持状态快捷切换（开始任务 / 完成任务 / 恢复待办）。
- 管理员角色：只读查看。

**UI-103 标签管理页**

- 管理员专用页面，从工作区配置进入。
- 展示当前工作区所有标签列表（名称 + 颜色）。
- 支持新建、编辑、删除标签操作。

## 6. 数据库/数据需求

### 6.1 总体策略

- 远端为事实源（Source of Truth）：Gitee 代码片段中存储工作区数据，所有读写直接对远端进行。
- 本地仅用于：
  - 保存当前用户角色和口令。
  - 保存当前选择的工作区标识。

### 6.2 远端数据结构（Gitee 代码片段）

一个工作区对应一个代码片段，包含以下文件（便于分拆与减少冲突面）：

- `meta.json`：工作区元信息、成员、标签与版本。
- `todos.json`：任务数据。

**DB-REMOTE-001 meta.json**

| 字段 | 类型 | 说明 |
|------|------|------|
| schemaVersion | int | 数据结构版本号 |
| workspace.workspaceId | UUID | 工作区唯一标识 |
| workspace.name | string | 工作区名称 |
| workspace.description | string | 工作区描述（可选） |
| workspace.createdAt | datetime | 创建时间 |
| workspace.updatedAt | datetime | 最后更新时间 |
| members[] | array | 成员列表（2-5 人） |
| members[].memberId | UUID | 成员唯一标识 |
| members[].displayName | string | 成员显示名 |
| members[].role | enum | 角色：admin / parent / student |
| tags[] | array | 标签列表（上限 20 个） |
| tags[].tagId | UUID | 标签唯一标识 |
| tags[].name | string | 标签名称（1-20 字） |
| tags[].color | string | 标签颜色（hex，可选） |
| tags[].createdAt | datetime | 创建时间 |
| revision.remoteRevision | string | 远端版本指纹（用于冲突检测） |

**DB-REMOTE-002 todos.json**

| 字段 | 类型 | 说明 |
|------|------|------|
| tasks[] | array | 任务列表 |
| tasks[].taskId | UUID | 任务唯一标识 |
| tasks[].title | string | 标题（必填，1-80 字） |
| tasks[].description | string | 详情描述（可选，1-256 字） |
| tasks[].status | enum | 状态：todo / in_progress / done |
| tasks[].priority | enum | 优先级：low / medium / high |
| tasks[].dueAt | datetime | 截止日期时间（必填） |
| tasks[].assigneeId | UUID | 指派人 memberId（只支持 1 个） |
| tasks[].tagIds | UUID[] | 关联标签 ID 数组（可选） |
| tasks[].startedAt | datetime | 开始时间（进行中时记录，可空） |
| tasks[].createdAt | datetime | 创建时间 |
| tasks[].createdBy | UUID | 创建人 memberId |
| tasks[].updatedAt | datetime | 最后更新时间 |
| tasks[].updatedBy | UUID | 最后更新人 memberId |
| tasks[].completedAt | datetime | 完成时间（可空） |
| tasks[].completedBy | UUID | 完成人 memberId（可空） |
| tasks[].deletedAt | datetime | 删除时间（可空，逻辑删除标记） |
| tasks[].deletedBy | UUID | 删除人 memberId（可空） |

### 6.3 数据一致性与并发控制

- 客户端写入远端时必须携带基线 revision，用于判断远端是否已被其他设备修改。
- 远端已变化时：
  - 先拉取最新远端数据。
  - 展示冲突提示，由用户决定覆盖或合并。
- 纯在线模式下每次操作立即同步，冲突概率较低。

## 7. 非功能需求

### 7.1 性能

- **NFR-001 启动性能**：冷启动进入任务列表不超过 3 秒（中端机）。
- **NFR-002 列表性能**：200 条任务滚动流畅，无明显卡顿。
- **NFR-003 接口响应**：正常网络下，单次 API 调用交互应在 2 秒内完成（不含网络抖动）。

### 7.2 可靠性与可用性

- **NFR-101 网络异常处理**：网络错误时明确提示用户检查网络连接，操作不丢失（即时重试）。
- **NFR-102 崩溃恢复**：崩溃后重启不丢失角色配置与当前工作区信息。

### 7.3 安全

- **NFR-201 敏感信息保护**：PAT、workspaceKey、口令必须使用系统安全存储加密保存。
- **NFR-202 传输安全**：与 Gitee API 通信必须走 HTTPS。
- **NFR-203 最小暴露**：应用日志不得输出 PAT、完整请求体中的敏感字段。

### 7.4 兼容性与可维护性

- **NFR-301 平台兼容**：Android/iOS 适配主流分辨率与深色模式（可选）。
- **NFR-302 可迁移**：支持将工作区数据迁移至新的 gistId（管理员操作，在 FR-003 工作区编辑功能中实现）。

### 7.5 易用性

- **NFR-401 核心闭环**：新增任务 → 指派 → 完成，路径不超过 3 个页面层级。
- **NFR-402 反馈明确**：网络状态、操作结果、冲突提示必须可见且可理解。

## 8. 验收标准（摘要）

- 管理员可创建工作区并在 Gitee 上生成对应的代码片段。
- 家长可创建、编辑、删除任务，学生可查看并处理分配给自己的任务。
- 管理员、家长、学生三种角色权限边界清晰，不可越权操作。
- 同一任务多端编辑能检测冲突，并提供可理解的提示。
- 所有数据存储于 Gitee 代码片段中，无需自建后端即可完整使用。
- 应用需联网使用，网络断开时有明确提示。
