# MyTodos团队待办App应用 — 设计规格

## 概述

一个面向 2-5 人小团队的移动端待办事项协作应用。使用 React Native 开发，通过 Gitee 代码片段 API 存储数据，无需自建后端。

## 技术选型

| 层面   | 选择                             | 理由                       |
| ---- | ------------------------------ | ------------------------ |
| 框架   | React Native                   | 跨平台，一套代码支持 iOS 和 Android |
| 数据存储 | Gitee 代码片段 API (v5/gists)      | 无需自建后端，自带版本历史            |
| 本地缓存 | AsyncStorage                   | React Native 内置，轻量够用     |
| 本地通知 | react-native-push-notification | 纯本地提醒，不依赖推送服务器           |
| 导航   | React Navigation               | 底部 Tab 导航                |
| 认证   | Gitee Personal Access Token    | 每个成员用自己 的 token          |

## 数据模型

整个团队共用一个 Gitee 代码片段，内部存储 `todos.json`：

```json
{
  "team": "团队名称",
  "members": ["alice", "bob", "carol"],
  "todos": [
    {
      "id": "uuid-string",
      "title": "完成首页设计",
      "description": "包括配色方案和布局图",
      "status": "todo | in_progress | done",
      "priority": "low | medium | high",
      "assignee": "alice",
      "creator": "bob",
      "dueDate": "2026-04-10T00:00:00Z",
      "createdAt": "2026-04-02T10:00:00Z",
      "updatedAt": "2026-04-02T10:00:00Z",
      "tags": ["设计", "UI"]
    }
  ]
}
```

字段说明：

- `id` — UUID，全局唯一，避免多人操作冲突
- `status` — 三态：todo（待办）、in\_progress（进行中）、done（已完成）
- `priority` — low / medium / high
- `assignee` — 任务负责人，对应 Gitee 用户名
- `creator` — 创建者，对应 Gitee 用户名
- `dueDate` — 截止日期，ISO 8601 格式
- `tags` — 字符串数组，自定义标签用于筛选

### 模块职责

**Gitee API 客户端** — 封装三个核心操作：

- `fetchGist(id, token)` — 读取代码片段内容
- `updateGist(id, token, content)` — 更新代码片段内容
- `createGist(token, content)` — 创建新代码片段（首次初始化）

**数据服务层** — 核心业务逻辑：

- 读操作优先使用本地缓存
- 写操作先更新本地再同步到 Gitee
- 乐观锁机制（基于 `updatedAt` 时间戳检测冲突）
- 离线队列管理

**UI 层** — 4 个主页面，通过底部 Tab 导航切换。

**本地缓存** — AsyncStorage 存储完整 JSON 副本，支持离线查看。

## 页面设计

### 页面 1：任务列表（首页）

- 顶部筛选条：按状态/优先级/负责人快速切换
- 卡片式任务列表：每张卡片显示标题、优先级色标（红/黄/绿）、负责人、截止日期
- 下拉刷新同步 Gitee 数据
- 右下角浮动 "+" 按钮创建新任务
- 已完成任务灰显并置底

### 页面 2：任务详情/编辑

- 从任务列表点击卡片进入
- 表单字段：标题（必填）、描述、状态切换、优先级选择、指派给谁（下拉选择团队成员）、截止日期（日期选择器）、标签（可输入添加）
- 顶部操作栏：保存 / 删除
- 新建任务时复用此页面（空白表单）

### 页面 3：筛选

- 按状态筛选：全部 / 待办 / 进行中 / 已完成
- 按优先级筛选：高 / 中 / 低
- 按负责人筛选：团队成员列表
- 按标签筛选
- 筛选结果复用首页的卡片组件展示

### 页面 4：设置

- Gitee 私人令牌输入（密码模式显示）
- 团队代码片段 ID 配置（首次使用时输入）
- "初始化代码片段"按钮（自动创建新 gist 并填入空 JSON 模板）
- 团队成员管理（增删成员用户名）
- 手动同步按钮
- 关于/版本信息

## 同步策略

### 同步时机

| 触发场景       | 动作                    |
| ---------- | --------------------- |
| 打开 App     | 从 Gitee 拉取最新数据，更新本地缓存 |
| 下拉刷新       | 同上                    |
| 创建/编辑/删除任务 | 乐观更新：先写本地，立即同步到 Gitee |
| 网络恢复       | 自动推送本地未同步的变更          |

### 冲突检测与处理

策略：完整文件覆盖 + 时间戳校验。

1. 每次写入前，先 GET 最新 gist 内容，检查各 todo 的 `updatedAt` 时间戳
2. 如果远程数据比本地更新 — 弹窗提示用户选择：覆盖远程 或 合并到本地
3. 合并策略：逐条比较 todo 的 `updatedAt`，保留较新的版本
4. 如果无冲突 — 直接 PATCH 覆盖整个 JSON

### 离线支持

- 所有操作先写入本地 AsyncStorage
- 维护一个 `pendingActions` 队列，记录未同步的操作（类型、目标 todo id、时间戳、变更内容）
- 网络恢复后按顺序执行队列，逐条处理冲突
- 离线时 UI 显示"离线模式"标识

## 提醒功能

- 使用 react-native-push-notification 注册本地通知
- 每次同步数据时，对比 `dueDate` 与当前时间：
  - 明天到期的任务 — 注册一条提醒通知
  - 今天到期的任务 — 注册一条提醒通知
- 纯本地提醒，不依赖推送服务器
- 用户可在系统设置中关闭通知权限

## 认证流程

1. 用户在 Gitee 个人设置中生成私人令牌（勾选 projects 权限）
2. 在 App 设置页输入令牌
3. 令牌存储在 AsyncStorage 中，所有 API 请求通过 `access_token` 参数传递
4. 每个团队成员使用自己的令牌，操作记录可追溯到个人

## Gitee API 使用

使用的接口：

| 操作     | 方法    | 端点             |
| ------ | ----- | -------------- |
| 创建代码片段 | POST  | /v5/gists      |
| 获取代码片段 | GET   | /v5/gists/{id} |
| 更新代码片段 | PATCH | /v5/gists/{id} |
